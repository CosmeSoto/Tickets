🔵 GOOGLE OAUTH - RESUMEN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Ve a: https://console.cloud.google.com/
2. Crea proyecto: "Sistema de Tickets"
3. Habilita: Google+ API
4. Configura pantalla de consentimiento (Externo)
5. Crea credenciales OAuth 2.0:
   - Tipo: Aplicación web
   - Origen: http://localhost:3000
   - Redirect: http://localhost:3000/api/auth/callback/google
6. Copia Client ID y Secret

🔷 MICROSOFT OAUTH - RESUMEN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Ve a: https://portal.azure.com/
2. Azure Active Directory → Registros de aplicaciones
3. Nuevo registro: "Sistema de Tickets"
4. Tipos de cuenta: Multiinquilino y cuentas personales
5. Redirect URI: http://localhost:3000/api/auth/callback/azure-ad
6. Copia Application (Client) ID
7. Crea Client Secret en "Certificados y secretos"
8. Copia el Secret (¡solo se muestra una vez!)

🎯 CONFIGURAR EN TU SISTEMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Ve a: http://localhost:3000/admin/settings
2. Click en pestaña "OAuth"
3. Configura Google:
   - Pega Client ID
   - Pega Client Secret
   - ✅ Habilitar Google OAuth
   - Guardar
4. Configura Microsoft:
   - Pega Application (Client) ID
   - Pega Client Secret
   - Tenant ID: common
   - ✅ Habilitar Microsoft OAuth
   - Guardar

🚨 PROBLEMAS COMUNES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problema: Los botones no aparecen
Solución: Verifica que habilitaste los proveedores en la UI

Problema: "redirect_uri_mismatch"
Solución: Verifica que el URI sea exactamente el correcto

Problema: "invalid_client"
Solución: Verifica Client ID y Secret, cópialos nuevamente

Problema: "client_id is required"
Solución: Guarda la configuración en la UI y habilita el proveedor
